本博客基于 [Astro](https://astro.build) 构建，每篇文章都是一个标准的 Markdown 文件。以下介绍文件的格式要求和常用写作技巧。

## 文件格式

每一篇博客文章就是一个 `.md` 文件，放在 `src/content/posts/` 目录下。文件名会影响文章链接，例如 `my-first-post.md` 对应的 URL 是 `/posts/my-first-post`。

## Frontmatter

每篇文章文件**开头**必须有一段 YAML 格式的元数据，用 `---` 包裹，称为 Frontmatter：

```yaml
---
title: "文章标题"          # 必填
date: 2026-06-30            # 必填，YYYY-MM-DD 格式
description: "文章摘要"      # 可选，显示在首页卡片上
tags:                        # 可选，标签数组
  - 标签1
  - 标签2
draft: false                 # 可选，设为 true 则在生产环境隐藏
---
```

### 字段说明

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `title` | 是 | 字符串 | 文章标题，显示在页面顶部和首页卡片 |
| `date` | 是 | 日期 | 发布日期，格式 `YYYY-MM-DD` |
| `description` | 否 | 字符串 | 文章摘要，显示在首页卡片上 |
| `tags` | 否 | 字符串数组 | 标签，每行一个 `- 标签名` |
| `draft` | 否 | 布尔值 | 草稿标记，`true` 时不会出现在生产环境首页 |

### 注意

- Frontmatter **必须是文件的最开头**，前面不能有空行或注释
- `title` 中的特殊字符（如 `:`、`#`）需要用引号包裹
- `date` 不要加引号，直接写 `2026-06-30` 即可

## Markdown 基础语法

### 标题

```md
# 一级标题
## 二级标题
### 三级标题
```

`#` 的数量决定标题层级，**一级标题不要在正文中使用**（文章顶部已由 title 字段生成）。

### 段落与换行

```md
这是第一段。

这是第二段（空一行即新段落）。

这是同一段内的换行，
直接换行即可。
```

### 文字样式

```md
**粗体**
*斜体*
~~删除线~~
`行内代码`
```

### 链接

```md
[链接文字](https://example.com)
```

### 图片

```md
![替代文字](https://example.com/image.png)
```

图片建议使用图床外链，不要将图片文件放入仓库。

### 列表

无序列表：

```md
- 项目一
- 项目二
  - 嵌套子项
```

有序列表：

```md
1. 第一步
2. 第二步
3. 第三步
```

### 引用

```md
> 这是一段引用文字。
>
> 引用可以有多段。
```

### 代码块

用三个反引号包裹，并指定语言获得语法高亮：

````md
```python
def hello():
    print("Hello, World!")
```
````

常用语言标识：`js` `ts` `python` `bash` `json` `yaml` `html` `css` `md`

### 分隔线

```md
---
```

注意：分隔线 `---` 不能出现在 Frontmatter 之后紧邻的位置，否则会被解析为 Frontmatter 的结束标记。

### 表格

```md
| 姓名 | 年龄 | 城市 |
|------|------|------|
| 张三 |  25  | 北京 |
| 李四 |  30  | 上海 |
```

冒号控制对齐：

```md
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| a      |  b   |      c |
```

## 写作建议

1. **第一个二级标题**放在 Frontmatter 下方空一行之后
2. 正文中从 `##`（二级标题）开始使用，`#` 留给页面标题
3. 代码块指定语言以获得语法高亮
4. 适当使用空行分隔段落，提升可读性
5. 图片用外链，不要直接放入仓库

## 发布流程

1. 在 `src/content/posts/` 下创建 `.md` 文件
2. 填写 Frontmatter 和正文
3. `git push` 到 `main` 分支
4. 等待 GitHub Actions 自动构建部署（约 1-2 分钟）
5. 刷新 `https://bcggxx.github.io` 即可看到新文章
